# Privacy Policy

Last updated: April 20, 2026

Docs Drift Check is a local Codex skill and plugin for identifying likely documentation drift in repository changes.

## What the plugin does

- Reads repository diffs and nearby documentation files
- Highlights likely doc-sensitive changes such as env vars, routes, commands, config keys, and renames
- Produces a local summary for the user inside Codex

## Data handling

- Docs Drift Check is designed to run against local repositories or repositories you intentionally install it into.
- It does not include a hosted backend operated by the plugin author.
- It does not independently transmit repository contents to an external service controlled by the plugin author.

## Third-party platforms

When you use Docs Drift Check inside Codex, your usage remains subject to:

- OpenAI and Codex product terms and data handling
- Any services you separately connect to Codex
- Any hosting or CI systems you use around this repository

If you use GitHub, Vercel, or other services while working with this plugin, those services apply their own privacy terms.

## Analytics and tracking

- This repository and plugin do not include plugin-specific telemetry or analytics collection.
- If future versions add telemetry, this policy should be updated before that collection ships.
